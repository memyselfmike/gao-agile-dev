## Test Report Output Format

Provide a comprehensive test report with:

### 1. Test Results
- Test results (passing/failing counts)
- Specific test failures if any
- Performance metrics

### 2. Acceptance Criteria Verification
- [ ] Criterion 1 - PASS/FAIL with evidence
- [ ] Criterion 2 - PASS/FAIL with evidence
- [ ] Criterion 3 - PASS/FAIL with evidence
(List all acceptance criteria from story)

### 3. Code Quality Metrics
- Test coverage percentage
- Linting status (pass/fail)
- Type checking status (pass/fail)
- Code complexity assessment

### 4. Overall Verdict
- **Status**: PASS or FAIL
- **Summary**: Brief explanation of results
- **Issues**: If FAIL, specific issues that need fixing
- **Recommendation**: Ready to commit or needs rework
